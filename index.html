/ what it does

  • ▸ Lossless round-trips. Extract CBOR streams to JSON and repack back without data loss — bytes bodies are carried through as {"__cbor_bytes_b64":"…"} sentinels, and float precision is preserved end-to-end.
  • ▸ Streaming at scale. A single 3.3 MB export decompresses to 1.1 GB / 2000 records. Every read path streams record-by-record; nothing is buffered unless you ask for a JSON array.
  • ▸ Built-in redaction. Nine canned secret presets (email, jwt, apikey, bearer, aws, …) plus custom regexes, with one-pass strip-and-drop.
  • ▸ Split & merge. Break a stream into per-day or per-session files, or combine many exports into one — all CBOR→CBOR, no intermediate format.

/ commands

info <files>
Per-file summary: record counts, sizes, id/timestamp ranges, schema.
ls <files>
List records as a table or NDJSON, with filtering.
grep <pattern> <files>
Regex search across all string/bytes values in records.
verify <files>
Integrity-check: fully decode every record, report corruption.
extract <files>
Records to NDJSON or JSON array; project fields; dump bodies.
repack <input>
Re-encode JSON (NDJSON or array) back to .cbor.zstd.
edit <files>
Redact secrets, strip fields, drop/select — one pass.
stats <files>
Aggregate tokens, cost, durations; by model or path.
merge <files>
Combine many .cbor.zstd files into one (CBOR→CBOR).
split <files>
Split into per-group files: day / session / model / path.

/ shared filter flags

Every selection command (ls, extract, grep, edit, stats, merge, split) accepts these:

FlagMatches
--idrecord id or inclusive range (e.g. 5 or 5-10)
--modelexact model name
--providermodel prefix before /
--pathexact request path
--statusHTTP status code
--since / --untilISO-8601 time range (bare date = inclusive whole day)
--dateexact calendar day (YYYY-MM-DD)
--invertdrop matching records instead of keeping them
